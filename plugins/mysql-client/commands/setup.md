You are running `/mysql-client:setup`: an interactive configurator. You will write **only** `.claude/mysql-client.local.md` and (if needed) a line in `.gitignore`. You will **not** run the project's `connection_cmd`, **not** connect to any database, and **not** dispatch the `mysql-investigator` agent.

First read the shared checklist at `${CLAUDE_PLUGIN_ROOT}/setup-checklist.md` — it defines checks 1–6 and the remediation text. Use that text verbatim on failures.

## Flow

### 1. Prerequisite
Run check 1 (`mysql` client installed). If it fails, print its remediation text and **stop** — tell the user to re-run `/mysql-client:setup` once the client is installed.

### 2. Existing settings file?
Run check 2. If `.claude/mysql-client.local.md` already exists, Read it, show the current `connection_cmd`, and ask via **AskUserQuestion**: *Keep as-is* / *Reconfigure (overwrite)* / *Abort*. Stop on Keep or Abort.

### 3. Pick the secret-store flavor
**AskUserQuestion** — "How does this project fetch MySQL credentials?":

- **Vault** — "HashiCorp Vault: a secret holds either the whole [client] ini in one field, or discrete fields you assemble with jq."
- **AWS** — "AWS Secrets Manager (a JSON secret), or RDS IAM auth (a rotating 15-minute token)."
- **Encrypted file** — "A file decrypted on demand — SOPS, GnuPG, or the 1Password CLI."
- **Other** — "A plain local file, or any other command whose stdout is a [client] ini body."

### 4. Assemble `connection_cmd`
Open `${CLAUDE_PLUGIN_ROOT}/skills/mysql-client/references/local-settings.md` and show the user the template(s) matching their chosen flavor. Ask for the specifics — secret path, `jq` field paths, AWS region, file path, etc. — and assemble the final `connection_cmd`. Restate it and confirm before writing.

Enforce while assembling:

- The command's **stdout** must be the body of a `[client]`-section INI — not a connection URI, not a file path, not a status line.
- No `-p<password>` anywhere — the point is keeping passwords out of `ps`-visible argument lists. The `[client]` ini's `password=` line is fine; it is read from a file, not a process argument.
- Keep it task-agnostic. Per-target switches (dev/staging/prod) belong in an env var the command reads — `secret/db/${MYSQL_TARGET:-prod-reader}` — not hard-coded. Claude prepends `MYSQL_TARGET=…` per turn from the project's `CLAUDE.md`.

### 5. Write `.claude/mysql-client.local.md`
Create the `.claude/` directory if needed. Single-line command:

```markdown
---
# Generated by /mysql-client:setup. Edits take effect immediately — no restart.
# connection_cmd stdout must be the body of a [client]-section INI.
connection_cmd: <value>
---

# Project DB notes (for humans — not read by the plugin)

What this points at, which secret store, who to ping when it breaks.
```

Multi-line command — use a YAML block scalar:

```markdown
---
connection_cmd: |
  <line 1>
  <line 2>
---

# Project DB notes (for humans — not read by the plugin)
```

### 6. `.gitignore`
Run check 6. If `.gitignore` doesn't already cover the settings file, append `.claude/*.local.md` (create `.gitignore` if absent). If already covered, say so and change nothing.

### 7. Summary
Print the path written and the assembled `connection_cmd`. End with: "Done. The skill re-reads this file every turn — no restart needed. Run `/mysql-client:doctor` to verify the command resolves and the connection actually works."
