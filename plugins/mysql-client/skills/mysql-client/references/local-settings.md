# Example `.claude/mysql-client.local.md`

Drop this at `.claude/mysql-client.local.md` in the project root and add `.claude/*.local.md` to `.gitignore`. `/mysql-client:setup` writes the same shape interactively.

One frontmatter field — `connection_cmd` — a shell command whose **stdout is a `mysql://` (or `mariadb://`) URL**. The plugin runs the command, pipes the URL through its bundled converter (`scripts/mysql-url-to-cnf/`), and feeds the resulting `[client]` INI to `mysql --defaults-file`. You never write the INI yourself.

The file's markdown body is human notes; only the frontmatter is read by the plugin.

## Template

```markdown
---
# Generated by /mysql-client:setup. Edits take effect immediately — no restart.
# connection_cmd stdout must be a mysql:// or mariadb:// URL.
connection_cmd: <command>
---

# Project DB notes (for humans — not read by the plugin)

What this points at, which secret store, who to ping when it breaks.
```

## Example commands

### A make / build target that prints a URL

```yaml
connection_cmd: AWS_PROFILE=su-build make infra-mysql-url --no-print-directory ENV=dev
```

`--no-print-directory` keeps make's chatter off stdout; make sure the recipe itself is silent (`@`-prefixed) so **only** the URL is printed. Record the command exactly as you run it — concrete `ENV=dev` and all. The settings file is gitignored and per-developer, and the plugin re-reads it every turn, so switching to another environment is just editing this one line.

### A literal URL

```yaml
connection_cmd: echo 'mysql://reader:pw@db.internal:3306/app?sslmode=require'
```

There is no separate "static URL" field — wrap the literal in `echo`.

### Vault — secret holds the URL

```yaml
connection_cmd: vault read -field=url secret/db/prod-reader
```

### AWS Secrets Manager — secret value is the URL

```yaml
connection_cmd: aws secretsmanager get-secret-value --secret-id db/prod-reader --query SecretString --output text
```

### SOPS-encrypted file

```yaml
connection_cmd: sops -d ~/.config/mysql/prod-reader.url.sops
```

### 1Password CLI

```yaml
connection_cmd: op read "op://Engineering/db-prod-reader/url"
```

## What the URL needs

```
mysql://<user>:<password>@<host>[:<port>]/[<database>][?sslmode=<mode>]
```

- **scheme** — `mysql://` or `mariadb://`. Anything else is rejected.
- **user** — required.
- **password** — optional; omit the `:<password>` for a passwordless user.
- **host** — required.
- **port** — optional; defaults to `3306`.
- **database** — optional; sets the default schema.
- **`?sslmode=`** — optional; `disable` / `prefer` / `require` / `verify-ca` / `verify-full` map to MySQL's `--ssl-mode`. Other query parameters are ignored.

## How the URL becomes a connection

The bundled converter (`scripts/mysql-url-to-cnf/`) parses the URL into a `[client]` option-file block: it splits the userinfo on the **last** `@` (so a raw `@`, `:`, or `/` in the password survives), percent-decodes `user` / `password` / `database`, and double-quotes every value so a `#`, backslash, or space in the password can't corrupt the INI. A broken `connection_cmd` — empty output, make's recipe echo, anything that isn't a URL — makes the converter exit non-zero, so the failure surfaces instead of producing a garbage config.

URLs are assumed **percent-encoded** per the database-URL convention. The converter tolerates a raw `@` / `:` / `/` in a non-encoded password, but a literal `%` followed by two hex digits would be mis-decoded — percent-encode the password at the source if it can contain `%`.

## Security envelope

- `connection_cmd`'s stdout — the URL, password and all — is piped straight into the converter and on into a mode-600 tempfile. It **never enters the conversation transcript**.
- The tempfile is deleted at the end of every turn.
- `connection_cmd` runs in your shell with your env — write it as carefully as any other credential-handling shell.
- Stderr from `connection_cmd` and the converter may be shown; neither echoes the URL or password.
